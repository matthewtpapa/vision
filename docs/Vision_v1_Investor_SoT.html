Vision v1.0 — Investor Single Source of Truth

Status: Planning Locked • Release: v1.0 GA

CONFIDENTIAL

Vision is an offline, deterministic AR identification SDK for enterprises. It recognizes products and parts on-device with auditable performance, no cloud dependency in the recognition path, and a governed knowledge base. This document is the single source of truth: milestones, quantitative targets, and how we prove them.

Investor-Grade Guarantees

Deterministic & Private

Identical inputs → identical outputs; no data leaves device.

Operational Guarantees

End-to-end p95 ≤ 33 ms; unknown false-accept ≤ 2.5%.

Governed Knowledge

Explainable accept/reject and reversible knowledge updates.

What ships at v1.0: SDKs for Ubuntu (CPU), Jetson Orin (CUDA), and Android (NNAPI/CPU); a replay demo (video + event log) produced by CI; and a due-diligence evidence pack validated by CI.

Scope & Non-Goals

In scope: on-device detection, deterministic cropping, embedding, multi-signal fusion, verification, governed knowledge, reproducible builds, pilot kit.

Out of scope (v1.0): online services, training/fine-tuning on customer data, rich UI, tracking.

Product Overview

Inputs: RGB frames (camera or replay)

Pipeline: Decode → Detect → Crop → Embed → Fuse (vision + text + logo/OCR + context) → Verify → Evidence → Promotion

Outputs: ranked IDs with labels & scores, unknown flag, per-stage timings, provenance (crop IDs, boxes). All results are verified by CI using reproducible media packs.

Operating Model

A single CI entrypoint executes the roadmap twice in clean workspaces, checks targets, and publishes a signed summary. All claims in this document are supported by downloadable evidence in the build artifacts. If a target regresses, the release halts at that stage until corrected.

Pilot & Go-To-Market

Pilot: one vertical (e.g., retail shelf or industrial parts) with a curated knowledge shard (≤10k IDs). Success mirrors the end-to-end thresholds above.

Packaging: Ubuntu wheel, Jetson container, and Android AAR — each with a 10-line quickstart.

Commercial model: per-device annual license with support tiers; private knowledge curation available separately.

Evidence: zipped evidence pack (demo, event log, benches, purity, SBOM/licenses, promotion report, metrics hash).

Key Risks & Mitigations

  • Coverage gaps in text/linking → alias banks and logo/text priors blended by a learned fusion model.
  • Domain shift across cameras/scenes → calibration guards and vertical-specific medoids.
  • Build drift or unreliable tests → locked inputs, deterministic seeds, double-run verification.
  • License compliance → tracked model/data provenance and automated audits in CI.

Definition of Done (v1.0 GA)

All S1–S17 milestones validated in CI; SDKs built for Ubuntu, Jetson Orin, and Android; pilot evidence meets thresholds (p@1 ≥ 0.80, e2e p95 ≤ 33 ms, unknown false-accept ≤ 2.5%); compliance checks clean; pricing and sales materials committed.
